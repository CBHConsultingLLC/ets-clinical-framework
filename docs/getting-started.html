Installation

Method 1: Direct Download

Download the repository and include the necessary files in your project:

# Clone the repository
git clone https://github.com/yourusername/cerner-mpage-components.git

# Or download as ZIP
# Extract and copy the assets folder to your project

Method 2: CDN (Coming Soon)

Include the CSS and JavaScript files directly from CDN:

<!-- CSS -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/yourusername/cerner-mpage-components@latest/assets/css/main.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/yourusername/cerner-mpage-components@latest/assets/css/components.css">

<!-- JavaScript -->
<script src="https://cdn.jsdelivr.net/gh/yourusername/cerner-mpage-components@latest/assets/js/main.js"></script>

Basic Usage

1. Include Required Files

Add the CSS and JavaScript files to your mPage:

<!-- In your mPage head section -->
<link rel="stylesheet" href="assets/css/main.css">
<link rel="stylesheet" href="assets/css/components.css">
<link href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.0.0/css/all.min.css" rel="stylesheet">

<!-- Before closing body tag -->
<script src="assets/js/main.js"></script>

2. Use Component Classes

Wrap your content with the appropriate component classes:

<div class="mpage-component">
    <div class="workflow-container">
        <!-- Your workflow content -->
    </div>
</div>

3. Example: Simple Alert

Information
This is an example alert using the component system.
<div class="alert alert-info">
    <div class="alert-icon">
        <i class="fas fa-info-circle"></i>
    </div>
    <div class="alert-content">
        <div class="alert-title">Information</div>
        <div class="alert-message">This is an example alert using the component system.</div>
    </div>
</div>

Customization

CSS Custom Properties

All components use CSS custom properties for easy theming. Override these variables to match your organization's branding:

:root {
    --primary-color: #2c5aa0;      /* Main brand color */
    --secondary-color: #1e3a5f;    /* Secondary brand color */
    --accent-color: #4a90e2;       /* Accent color */
    --success-color: #28a745;      /* Success state color */
    --warning-color: #ffc107;      /* Warning state color */
    --danger-color: #dc3545;       /* Danger state color */
    --info-color: #17a2b8;         /* Info state color */
    --text-color: #333;            /* Main text color */
    --border-color: #dee2e6;       /* Border color */
    --border-radius: 8px;          /* Default border radius */
}

Component-Specific Customization

Each component can be customized by overriding specific CSS classes:

/* Customize workflow steps */
.workflow-step .step-number {
    background-color: var(--your-custom-color);
    color: white;
}

/* Customize alert styling */
.alert.alert-custom {
    background-color: #f8f9fa;
    border-left-color: var(--your-custom-color);
}

JavaScript Customization

Extend component functionality by adding custom event listeners:

// Custom workflow step handler
document.addEventListener('workflowStepChanged', function(event) {
    console.log('Step changed to:', event.detail.stepIndex);
    // Your custom logic here
});

// Custom alert handler
document.addEventListener('alertDismissed', function(event) {
    console.log('Alert dismissed:', event.detail.alertType);
    // Your custom logic here
});

Best Practices

Performance

  • Minify CSS and JavaScript files for production
  • Use lazy loading for large component libraries
  • Optimize images and icons
  • Implement proper caching strategies

Accessibility

  • Always provide alternative text for images
  • Use semantic HTML elements
  • Ensure proper color contrast ratios
  • Implement keyboard navigation support
  • Test with screen readers

Security

  • Validate all user inputs
  • Sanitize data before display
  • Use HTTPS in production
  • Implement proper authentication
  • Follow HIPAA compliance guidelines

Code Organization

  • Keep components modular and reusable
  • Use consistent naming conventions
  • Document complex functionality
  • Implement proper error handling
  • Write unit tests for critical functions

Troubleshooting

Common Issues

Components not displaying correctly

Cause: Missing CSS files or incorrect file paths

Solution: Verify that all CSS files are included and paths are correct

JavaScript functionality not working

Cause: JavaScript file not loaded or DOM not ready

Solution: Ensure JavaScript is loaded and wrapped in DOMContentLoaded event

Styling conflicts with existing CSS

Cause: CSS specificity issues or conflicting styles

Solution: Use more specific selectors or CSS custom properties

Responsive design issues

Cause: Missing viewport meta tag or CSS media queries

Solution: Ensure viewport meta tag is present and CSS is responsive
